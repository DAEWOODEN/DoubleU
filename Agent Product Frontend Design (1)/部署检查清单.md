# ComChatX 部署检查清单

使用此清单逐步完成部署过程。每完成一步，请在方框中打勾 ✅。

## 📦 前置准备

- [ ] 已安装 Node.js（运行 `node -v` 检查，应该是 v18 或更高版本）
- [ ] 已安装 Git（运行 `git --version` 检查）
- [ ] 已有 GitHub 账号
- [ ] 已有 Vercel 账号（可使用 GitHub 登录）
- [ ] 域名 `comchatx.me` 已在阿里云购买

## 🔧 第一步：本地测试

- [ ] 已在项目目录中运行 `npm install`
- [ ] 已安装缺失的依赖：`npm install --save-dev typescript @types/react @types/react-dom`
- [ ] 本地开发服务器已启动（`npm run dev`）
- [ ] 浏览器可以正常访问 `http://localhost:3000`
- [ ] 构建测试通过（`npm run build` 成功）

## 📤 第二步：GitHub 设置

- [ ] 已在 GitHub 创建新仓库（例如：`comchatx-frontend`）
- [ ] 已在本地初始化 Git：`git init`
- [ ] 已创建第一次提交：`git commit -m "Initial commit"`
- [ ] 已连接到 GitHub：`git remote add origin <仓库地址>`
- [ ] 代码已推送到 GitHub：`git push -u origin main`
- [ ] GitHub 仓库中可以看到所有项目文件

## 🚀 第三步：Vercel 部署

- [ ] 已登录 Vercel 并授权 GitHub 访问
- [ ] 已在 Vercel 中导入 GitHub 仓库
- [ ] 已确认项目设置正确：
  - Framework: Vite
  - Build Command: `npm run build`
  - Output Directory: `dist`
- [ ] 首次部署已完成
- [ ] Vercel 临时域名可以正常访问（例如：`comchatx-frontend.vercel.app`）

## 🌐 第四步：域名配置

### Vercel 端

- [ ] 已在 Vercel 项目设置中添加域名：`comchatx.me`
- [ ] 已记录 Vercel 显示的 DNS 配置要求

### 阿里云 DNS 配置

- [ ] 已登录阿里云域名控制台
- [ ] 已找到域名 `comchatx.me` 的解析设置
- [ ] 已添加 CNAME 记录（或 A 记录，根据 Vercel 要求）：
  - 主机记录：`@`
  - 记录值：Vercel 提供的值
- [ ] 已添加 www 子域名记录（可选）：
  - 主机记录：`www`
  - 记录值：Vercel 提供的值

### DNS 验证

- [ ] Vercel Domains 页面显示域名状态为 ✅ 已验证
- [ ] 可以使用 `nslookup comchatx.me` 或 `dig comchatx.me` 检查 DNS

## ✅ 第五步：最终验证

- [ ] 可以通过 `https://comchatx.me` 访问网站
- [ ] 网站显示 HTTPS 安全连接（锁图标）
- [ ] 页面内容正常显示
- [ ] 交互功能正常工作
- [ ] 已测试移动端访问（可选）

## 🎯 完成

- [ ] 所有步骤已完成
- [ ] 网站已成功部署并可通过域名访问

---

## 📝 重要提示

1. **DNS 传播时间**：域名配置后，可能需要 10 分钟到 48 小时才能在全球生效，通常 10-30 分钟内生效。

2. **自动部署**：后续每次将代码推送到 GitHub，Vercel 会自动重新部署：
   ```bash
   git add .
   git commit -m "更新描述"
   git push
   ```

3. **遇到问题**：
   - 查看详细步骤：打开 `DEPLOYMENT_GUIDE.md`
   - 检查 Vercel 构建日志
   - 检查 DNS 配置是否正确
